# iOS プラットフォームガイド

`apple_design_principles.md` と併用する。本文書は iOS 固有のデザイン指針に特化する。

---

## 1. 入力モデル: タッチ操作

### タッチターゲット

- **最小サイズ**: 44 x 44 pt（これより小さいとタップエラー率が 25% 以上増加）
- **推奨サイズ**: 48 x 48 pt 以上（Fat Finger Problem 対策）
- **タップ領域の拡張**: 視覚的には小さくても、hit area は最低 44pt を確保する

### ジェスチャー

| ジェスチャー | 用途 | HIG 標準 |
|------------|------|---------|
| タップ | 主要操作 | ボタン、セル選択 |
| ロングプレス | コンテキストメニュー | メニュー表示、プレビュー |
| スワイプ（左右） | ナビゲーション、行アクション | 戻る、削除/アーカイブ |
| スワイプ（上下） | スクロール | コンテンツ閲覧 |
| ピンチ | ズーム | 画像・地図の拡大縮小 |
| 3D Touch / Haptic Touch | クイックアクション | ホーム画面ショートカット |

### Reachability（片手操作考慮）

- **親指ゾーン**: 画面下部 1/3 が最も操作しやすい
- **重要な操作は下部に配置**: Tab Bar、FAB、主要アクションボタン
- **画面上部は表示に適する**: タイトル、ステータス情報
- 大画面デバイス（iPhone Pro Max 系）では特に重要

## 2. ナビゲーションパターン

### Tab Bar（タブバー）

- **配置**: 画面最下部（固定）
- **タブ数**: 最大 5 個（5 個を超える場合は「その他」タブ）
- **各タブ**: アイコン + ラベル（アイコンのみは避ける）
- **SF Symbols**: タブアイコンは SF Symbols を使用
- **選択状態**: アクセントカラーで強調
- **永続性**: アプリ全体で常に表示

**UX 根拠**: タブバーは iOS ユーザーが最も慣れたナビゲーションパターン。画面下部配置は親指ゾーンと一致する。

### Navigation Bar（ナビゲーションバー）

- **配置**: 画面最上部
- **構成**: 戻るボタン（左）、タイトル（中央）、アクションボタン（右）
- **Large Title**: コンテンツ一覧画面で使用、スクロールで標準サイズに遷移
- **戻るジェスチャー**: 左端からの右スワイプで「戻る」を常にサポート

### Modal（モーダル）

- **用途**: 独立したタスク、確認ダイアログ
- **表示スタイル**: Sheet（部分表示）、Full Screen（全画面）
- **閉じ方**: 下スワイプ、「完了」/「閉じる」ボタン
- **注意**: モーダルの多用はフローを分断する。必要最小限に。

### 画面遷移

| パターン | 用途 | アニメーション |
|---------|------|-------------|
| Push | 階層を掘り下げる | 左からスライドイン |
| Modal Sheet | 独立タスク | 下から浮き上がり |
| Full Screen Modal | 没入タスク | 下からフルカバー |
| Tab Switch | トップレベル切替 | フェード / なし |

## 3. レイアウト

### Safe Area

- **上部**: ステータスバー + Dynamic Island / ノッチ
- **下部**: ホームインジケータ
- **コンテンツは Safe Area 内に配置**: 背景色・画像は Safe Area 外まで拡張してよい
- **横向き**: 左右にもSafe Area が発生する

### 画面サイズ対応

| デバイス | 論理幅 (pt) | 考慮事項 |
|---------|-----------|---------|
| iPhone SE | 375 | 最小画面。情報密度を抑える |
| iPhone 標準 | 390-393 | 基準デザインサイズ |
| iPhone Pro Max | 430 | 片手操作の限界。Reachability 重要 |
| iPad | 744-1024 | マルチカラムレイアウト検討 |

### スペーシング標準

| 用途 | 推奨値 |
|------|-------|
| 画面端マージン | 16pt（コンパクト幅）/ 20pt（レギュラー幅） |
| セクション間 | 24-32pt |
| 要素間（関連） | 8-12pt |
| 要素間（無関連） | 16-24pt |
| カード内パディング | 16pt |
| リストセル高さ | 44pt（最小） |

## 4. iOS 標準コンポーネント

デザイン分析時、以下の HIG 標準コンポーネントとの対応付けを行う。

### コントロール

| コンポーネント | 用途 | HIG 注意点 |
|-------------|------|-----------|
| Button | 操作トリガー | テキスト or アイコン。bordered / borderless / filled |
| Toggle | ON/OFF 切替 | 即座に反映。確認不要 |
| Slider | 連続値の調整 | 最小/最大の意味をアイコンで表示 |
| Picker | 値の選択 | Wheel / Segmented / Menu |
| Stepper | 数値の増減 | +/- ボタン |
| TextField | テキスト入力 | プレースホルダー、クリアボタン |
| SearchBar | 検索入力 | キャンセルボタン付き |

### コンテンツ表示

| コンポーネント | 用途 |
|-------------|------|
| List | 行ベースのコンテンツ一覧（Static / Dynamic） |
| Grid | グリッドベースのコンテンツ表示 |
| Card | 関連情報のグループ化 |
| Alert | 重要な通知、確認ダイアログ |
| ActionSheet | コンテキスト依存のアクション選択 |
| ProgressView | 進捗表示（線形 / 円形） |
| Badge | 通知数・ステータス |

### ナビゲーション

| コンポーネント | 用途 |
|-------------|------|
| TabBar | トップレベルナビゲーション |
| NavigationBar | 階層ナビゲーション |
| ToolBar | コンテキスト操作 |
| SegmentedControl | ビュー切り替え |

## 5. iOS 固有の UX パターン

### プルトゥリフレッシュ

- スクロールコンテンツの更新に使用
- スピナーアニメーションでフィードバック

### スワイプアクション

- リストセルの左スワイプ: 削除・アーカイブ等の破壊的/二次的アクション
- リストセルの右スワイプ: ピン留め・未読等の非破壊的アクション
- 色で操作の性質を示す（赤 = 削除、青 = 情報系）

### Context Menu（コンテキストメニュー）

- ロングプレスでトリガー
- プレビュー + アクションリスト
- SF Symbols アイコン付きのメニュー項目

### Haptic Feedback（触覚フィードバック）

| 種別 | 用途 |
|------|------|
| Impact | 物理的な接触感（ボタンタップ） |
| Selection | 選択肢の変更（ピッカー回転） |
| Notification | 成功/警告/エラーの通知 |

## 6. iOS 評価チェックリスト

### タッチ操作

- [ ] 全てのインタラクティブ要素が 44x44pt 以上か
- [ ] 近接する操作要素間に十分な間隔があるか
- [ ] 主要操作が親指ゾーン（画面下部）に配置されているか
- [ ] ジェスチャーが HIG 標準に従っているか

### ナビゲーション

- [ ] Tab Bar が 5 タブ以内か
- [ ] Tab Bar にアイコンとラベルの両方があるか
- [ ] 戻るジェスチャー（左端スワイプ）が機能するか
- [ ] Large Title の使用が適切か

### レイアウト

- [ ] Safe Area を尊重しているか
- [ ] Dynamic Island / ノッチとの干渉がないか
- [ ] スペーシングが 8pt グリッドに従っているか
- [ ] 最小画面（iPhone SE: 375pt）で情報が収まるか

### アクセシビリティ

- [ ] Dynamic Type（フォントサイズ変更）に対応しているか
- [ ] VoiceOver でナビゲーション可能か
- [ ] Reduce Motion 設定を尊重しているか
