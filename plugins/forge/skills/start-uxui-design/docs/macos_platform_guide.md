# macOS プラットフォームガイド

`apple_design_principles.md` と併用する。本文書は macOS 固有のデザイン指針に特化する。

---

## 1. 入力モデル: ポインタ + キーボード

### ポインタ操作

- **精度**: ピクセル単位の精密なポインティングが可能（タッチより高精度）
- **クリックターゲット**: 最小推奨は 20x20pt（iOS の 44pt より小さくてよい）
- **ホバー状態**: マウスオーバーで視覚的フィードバックを提供（iOS にはない概念）
- **右クリック**: コンテキストメニューの標準トリガー
- **ドラッグ & ドロップ**: ファイル操作、並べ替え、ウィンドウ間のデータ移動

### カーソル形状

| カーソル       | 意味                      |
| -------------- | ------------------------- |
| 矢印           | デフォルト                |
| 手 (pointing)  | クリック可能なリンク      |
| I-beam         | テキスト選択可能          |
| 十字           | 範囲選択                  |
| リサイズ       | ウィンドウ/要素のリサイズ |
| 待機 (spinner) | 処理中                    |

### キーボード操作

macOS ではキーボード操作が**第一級の入力手段**。パワーユーザーはキーボードだけで全操作を完結できることを期待する。

#### 標準ショートカット（必須）

| ショートカット        | 操作                       |
| --------------------- | -------------------------- |
| Cmd+C / Cmd+V / Cmd+X | コピー / ペースト / カット |
| Cmd+Z / Cmd+Shift+Z   | 取り消し / やり直し        |
| Cmd+A                 | 全選択                     |
| Cmd+S                 | 保存                       |
| Cmd+N                 | 新規                       |
| Cmd+W                 | ウィンドウを閉じる         |
| Cmd+Q                 | アプリを終了               |
| Cmd+,                 | 設定を開く                 |
| Cmd+F                 | 検索                       |
| Cmd+P                 | 印刷                       |

#### カスタムショートカット

- **Cmd + 英字**: 主要な独自操作
- **Cmd+Shift + 英字**: 二次的操作
- **Cmd+Option + 英字**: 高度な操作
- 既存の標準ショートカットとの衝突を避ける

## 2. ナビゲーションパターン

### メニューバー（必須）

macOS アプリの最も重要な要素。**すべてのアクションにメニューバーからアクセスできるべき**。

#### 標準メニュー構成

| メニュー | 内容                                       |
| -------- | ------------------------------------------ |
| アプリ名 | About、設定、サービス、非表示、終了        |
| File     | 新規、開く、保存、閉じる                   |
| Edit     | 取り消し、やり直し、コピー、ペースト、検索 |
| View     | 表示オプション、ツールバー表示/非表示      |
| Window   | 最小化、ズーム、全画面                     |
| Help     | ヘルプ検索、ドキュメント                   |

#### メニュー設計原則

- 全メニュー項目にキーボードショートカットを割り当てる
- 現在の状態で無効な項目はグレーアウト（非表示にしない）
- チェックマーク / ダッシュでトグル状態を表示
- セパレータで論理的にグループ化

### ツールバー

- **配置**: ウィンドウ上部（タイトルバー統合可能）
- **内容**: よく使うアクションのボタン、検索フィールド
- **カスタマイズ**: ユーザーがツールバー項目を並べ替え・追加・削除できるべき
- **アイコンサイズ**: 標準 / コンパクトの切替を提供

### サイドバー

- **配置**: ウィンドウ左端
- **用途**: ソースリスト（メール、Finder 等）、ナビゲーション階層
- **開閉**: ユーザーが表示/非表示を切り替えられるべき
- **幅**: リサイズ可能、適切な最小幅を設定
- **セクション**: 折り畳み可能なグループで構成

### インスペクタ

- **配置**: ウィンドウ右端
- **用途**: 選択要素の詳細表示・プロパティ編集
- **開閉**: ユーザーが表示/非表示を切り替えられるべき

### タブ

- **ドキュメントタブ**: 複数ドキュメントの切り替え（Safari, Finder）
- **セグメントタブ**: コンテンツビューの切り替え
- 新規タブの追加・並べ替え・閉じる操作をサポート

## 3. ウィンドウ管理

### ウィンドウの基本

- **マルチウィンドウ**: 複数ウィンドウの同時表示が標準
- **リサイズ**: 全方向からのリサイズを許可
- **最小サイズ**: コンテンツが破綻しない最小サイズを設定
- **ウィンドウ復元**: 前回終了時のウィンドウ位置・サイズを記憶する

### ウィンドウスタイル

| スタイル       | 用途                                             |
| -------------- | ------------------------------------------------ |
| 標準ウィンドウ | 一般的なドキュメント・アプリケーションウィンドウ |
| パネル（HUD）  | ユーティリティ、インスペクタ（最前面に浮く）     |
| Sheet          | ウィンドウに付随する確認ダイアログ               |
| Popover        | コンテキスト情報の表示                           |
| Alert          | 重要な確認（モーダル）                           |

### フルスクリーン

- **Split View**: 2つのアプリを左右に並べて表示
- **Stage Manager**: ウィンドウグループの管理
- フルスクリーン時のメニューバーは自動非表示（マウスを上端に移動で表示）

### レイアウトパターン

| パターン       | 構成                                       | 例                |
| -------------- | ------------------------------------------ | ----------------- |
| ソースリスト   | サイドバー + コンテンツ                    | Finder, Mail      |
| ブラウザ       | 複数カラム                                 | Finder カラム表示 |
| キャンバス     | ツールバー + 広いキャンバス + インスペクタ | Keynote, Xcode    |
| ユーティリティ | 単一ウィンドウ、コンパクト                 | 計算機            |

## 4. レイアウト

### 情報密度

macOS は iOS に比べて**高い情報密度**が許容される。

- 大画面・高解像度を活かした詳細表示
- テーブル/リストの行高は iOS より小さくてよい（24-28pt 程度）
- ツールバー・サイドバーでコンテキスト情報を常時表示可能
- ただし情報過多は避ける（ゲシュタルトの近接原則を活用してグループ化）

### スペーシング標準

| 用途                 | 推奨値                  |
| -------------------- | ----------------------- |
| ウィンドウ内マージン | 20pt                    |
| セクション間         | 20-24pt                 |
| 要素間（関連）       | 6-8pt                   |
| 要素間（無関連）     | 12-16pt                 |
| テーブル行高         | 24-28pt                 |
| サイドバー幅         | 200-250pt（デフォルト） |

### テキストサイズ

macOS は iOS よりも小さいフォントサイズが標準的に使用される。

| スタイル | iOS  | macOS |
| -------- | ---- | ----- |
| Body     | 17pt | 13pt  |
| Title    | 28pt | 22pt  |
| Caption  | 12pt | 10pt  |

## 5. macOS 標準コンポーネント

デザイン分析時、以下の HIG 標準コンポーネントとの対応付けを行う。

### コントロール

| コンポーネント      | 用途                       | macOS 固有の注意点                               |
| ------------------- | -------------------------- | ------------------------------------------------ |
| Push Button         | 操作トリガー               | ラベル付き。アイコンのみは避ける                 |
| Pop-up Button       | 値の選択（ドロップダウン） | iOS の Picker に相当                             |
| Checkbox            | ON/OFF 切替                | iOS の Toggle に相当（macOS はチェックボックス） |
| Radio Button        | 排他的選択                 | iOS にはない。Segmented Control で代替される     |
| Combo Box           | テキスト入力 + 選択        | 自由入力と候補選択の組合せ                       |
| Date Picker         | 日付/時刻選択              | カレンダー表示                                   |
| Disclosure Triangle | 詳細の展開/折り畳み        | ツリー構造の開閉                                 |

### コンテンツ表示

| コンポーネント  | 用途                                         |
| --------------- | -------------------------------------------- |
| Table View      | 列ベースのデータ表示（ソート・リサイズ可能） |
| Outline View    | ツリー構造のデータ表示                       |
| Collection View | グリッドベースのコンテンツ表示               |
| Split View      | 分割ペイン（サイドバー + コンテンツ等）      |
| Scroll View     | スクロール可能なコンテンツ領域               |
| Token Field     | タグ入力（メール宛先等）                     |

### ナビゲーション

| コンポーネント | 用途                          |
| -------------- | ----------------------------- |
| Menu Bar       | アプリ全体のコマンドアクセス  |
| Toolbar        | よく使うアクションの集約      |
| Sidebar        | ソースリスト / ナビゲーション |
| Tab View       | コンテンツビューの切り替え    |
| Path Control   | パス（パンくずリスト）        |

## 6. macOS 固有の UX パターン

### システム連携

- **Spotlight**: アプリ内検索をSpotlight と連携
- **Dock メニュー**: Dock アイコンの右クリックメニューにクイックアクションを提供
- **Shortcuts**: Shortcuts アプリからのアクション実行に対応
- **通知**: Notification Center への通知送信
- **Share Extension**: 共有メニューへの統合

### ドラッグ & ドロップ

macOS では**ドラッグ & ドロップは第一級のインタラクション**。

- ウィンドウ間・アプリ間のドラッグ & ドロップをサポート
- ドラッグ中のビジュアルフィードバック（プレビュー、ドロップ可能位置のハイライト）
- ファイルのインポート・エクスポートはドラッグ & ドロップで完結すべき

### マルチセレクション

- **Cmd+クリック**: 個別選択の追加/解除
- **Shift+クリック**: 範囲選択
- **Cmd+A**: 全選択
- 選択状態の明確な視覚的フィードバック

### Undo/Redo

macOS ユーザーは強力な Undo/Redo を期待する。

- **Cmd+Z / Cmd+Shift+Z**: 複数段階の Undo/Redo
- 操作履歴の管理
- 可能な限りすべての操作を取り消し可能にする

## 7. macOS 評価チェックリスト

### ポインタ操作

- [ ] ホバー状態が適切に表現されているか
- [ ] 右クリック（コンテキストメニュー）がサポートされているか
- [ ] ドラッグ & ドロップが適切な箇所で機能するか
- [ ] カーソル形状が操作の種類を正しく示しているか

### キーボード

- [ ] 標準ショートカット（Cmd+C/V/Z 等）が実装されているか
- [ ] Tab キーでフォーカス移動ができるか
- [ ] 全操作がキーボードのみで完結できるか
- [ ] ショートカットがメニューバーに表示されているか

### メニューバー

- [ ] 標準メニュー構成（File / Edit / View / Window / Help）に従っているか
- [ ] 全アクションにメニューバーからアクセスできるか
- [ ] 無効な項目がグレーアウトされているか（非表示ではなく）
- [ ] メニュー項目にキーボードショートカットが割り当てられているか

### ウィンドウ管理

- [ ] ウィンドウがリサイズ可能か
- [ ] 最小サイズが適切に設定されているか
- [ ] ウィンドウ位置・サイズが終了時に記憶されるか
- [ ] フルスクリーン / Split View に対応しているか

### 情報密度

- [ ] 画面の広さを活かした情報表示ができているか
- [ ] テーブル・リストのカラムがソート・リサイズ可能か
- [ ] サイドバー/インスペクタの表示/非表示が切り替えられるか
- [ ] 情報過多になっていないか（ゲシュタルト原則の活用）

### アクセシビリティ

- [ ] VoiceOver でナビゲーション可能か
- [ ] フルキーボードアクセスが可能か
- [ ] Reduce Motion 設定を尊重しているか
- [ ] コントラスト比が 4.5:1 以上か
