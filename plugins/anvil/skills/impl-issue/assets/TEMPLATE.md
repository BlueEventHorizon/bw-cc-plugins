# Issue 更新テンプレート

以下のテンプレートを使用して Issue 本文を更新する。
実装スコープに応じて不要なセクションを削除すること。

**書き込む内容**: 解決内容（対策・実装計画・TODO）のみ追記する。
背景/現象・原因はすでに記載済みのため、上書きしない。

## テンプレート

```markdown
## 対策

<解決方針・具体的なアプローチ>

## 実装計画

### 関連ドキュメント

**仕様書**

| 種類   | ドキュメント    |
| ------ | --------------- |
| {種類} | [{名前}]({URL}) |

**参考PR（実装パターン）**

| PR                        | 参考ポイント             |
| ------------------------- | ------------------------ |
| [#<番号> <タイトル>](URL) | <参考にした実装パターン> |

### 実装スコープ

| レイヤー  | 実装内容              | ファイルパス |
| --------- | --------------------- | ------------ |
| {Layer名} | {クラス名 / 変更内容} | `{path}`     |

### スコープ外

| 項目 | 理由 | 担当 |
| ---- | ---- | ---- |

## TODO

### 着手前確認

- [ ] 仕様書の確認完了
- [ ] 実装ルールの確認完了
- [ ] 参考PRのパターン確認完了

### 実装する

- [ ] {実装タスク 1}（`{path}`）
- [ ] {実装タスク 2}（`{path}`）

### 確認・レビュー準備を行う

- [ ] ビルド成功確認
- [ ] 静的解析・lint 通過
- [ ] テスト実行（存在する場合）

### 完了する

- [ ] PR を作成する
- [ ] レビューに対応する
```

## gh コマンド例

```bash
# 本文を一時ファイルに書き出す（特殊文字・バッククォート・ドル記号が含まれても安全）
tee /tmp/issue_body.md <<'BODY'
<上記テンプレートの内容>
BODY

# --body-file で渡す（<owner>/<repo> は Phase 0 で取得した値を使用）
gh issue edit <issue番号> --repo <owner>/<repo> --body-file /tmp/issue_body.md
```
