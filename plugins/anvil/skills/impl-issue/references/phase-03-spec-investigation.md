# Phase 3: 仕様書調査ルール

`/doc-advisor:query-specs` を使い、Issue のタイトル・本文から **抽出した検索キーワード** または **短い自然文のタスク記述** を `args` として渡し、関連仕様書を特定する。

## args の渡し方 [MANDATORY]

COMMON-DES-001 §4 に従い、`args` は以下に限定する:

- ✅ Issue タイトル / 本文から抽出した短いキーワード（例: `"Repository 実装パターン"`, `"ログイン画面 ViewModel"`）
- ✅ 短い自然文のタスク記述（例: `"ログイン画面の状態遷移を実装したい"`）
- ❌ Issue 本文の全文貼り付け
- ❌ 「上記タスクに関連する仕様書を検索してください」のような指示連結

## 必須ルール

- 特定した仕様書は**すべて**実際に Read tool で読み込む（Grep 検索だけで済ませない）
- API レスポンス構造、画面要件、制約・バリデーションルールを完全に把握する
- 関連する可能性があるドキュメントも積極的に読む

## Issue 参照ルール

Issue に記載する参照は **GitHub / Figma でブラウザから開けるものだけ** にする：

- commit & push 済みで GitHub 上に存在するファイルは GitHub blob URL で記載する
- ローカルにしか存在しないファイルは Issue の関連ドキュメント欄には載せない
- Figma URL はそのまま記載する

## UI Issue の場合の追加作業

画面設計書と確認・調整事項（存在する場合）を必ず読み、以下を記録する（Phase 7 で prepare-figma に渡す）：

- 画面設計書ファイルパス
- Figma URL（画面設計書に記載されているもの）
- 画面 ID・画面名
- 確認・調整事項ファイルパス（存在する場合）
