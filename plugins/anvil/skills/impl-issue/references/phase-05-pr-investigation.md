# Phase 5: 類似PR調査ルール

今回の実装と**同じスコープ**（同様の機能・同様のレイヤー変更）のマージ済みPRを3件以上探す。

## コマンド例

```bash
# キーワードで検索
gh pr list --repo <owner>/<repo> --state merged --search "<キーワード>" --limit 20

# PRの詳細確認
gh pr view <pr番号> --repo <owner>/<repo>

# 変更ファイル一覧確認
gh pr diff <pr番号> --repo <owner>/<repo> --name-only
```

> `<owner>/<repo>` は Phase 0 で取得したリポジトリ名を使用する。

## 調査のポイント

- どのレイヤー・ファイルが変更されているか
- ドメインモデルの実装パターン
- リポジトリ・サービスの実装パターン
- DI 設定方法（依存注入・ファクトリの構成）
- 状態管理の実装スタイル

## 原則

今回の実装は調査したPRと**同じスコープ・同じ実装方法**を採用する。
