# FNC-005: doc-db Index 生成 機能要件

## 概要

ドキュメントを見出し単位の chunk に分割し、各 chunk を Embedding ベクトルに変換してローカル Index ファイルに永続化する機能。検索時にこの Index を読み込んで類似度計算を行う。

## 前提条件

- `.doc_structure.yaml` が定義済みである
- OpenAI API key が利用可能である
- 対象ドキュメントは Markdown 形式（`.md`）である

## 要件一覧

### 操作要件

| ID    | 要件                                                                                              |
| ----- | ------------------------------------------------------------------------------------------------- |
| OP-01 | ユーザーは Index 生成 skill を実行することで、対象ディレクトリ配下の全 Markdown を Index 化できる |
| OP-02 | ユーザーは差分更新を選択でき、変更があった文書のみを再 Embedding できる                           |
| OP-03 | ユーザーは Index ファイルの保存先を指定できる                                                     |

### Chunk 分割要件

| ID     | 要件                                                                                         |
| ------ | -------------------------------------------------------------------------------------------- |
| CHK-01 | 1 chunk は Markdown の見出し境界で切り出す                                                   |
| CHK-02 | chunk には所属する文書パスと、文書内での見出し階層パス（例: `# A > ## B > ### C`）を保持する |
| CHK-03 | 見出しを持たない文書は、文書全体を 1 chunk として扱う                                        |
| CHK-04 | chunk が極端に長い場合の分割上限（token 数）はコードデフォルトで定義する                     |

### Embedding 要件

| ID     | 要件                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------ |
| EMB-01 | Embedding により chunk を意味ベクトル化する（モデル・次元数は設計書で確定。参考実装: OpenAI `text-embedding-3-large`） |
| EMB-02 | 1 文書は複数のベクトル（chunk ごと、必要に応じて field ごと）を持てる（Multi-vector / Field 別） |
| EMB-03 | 外部 pip パッケージ（OpenAI SDK 等）を必須としない手段で Embedding API を呼び出す（HTTP クライアントは設計書で確定） |
| EMB-04 | API 呼び出しは複数 chunk をバッチ化して効率化する（バッチサイズは設計書で確定）                  |

### 永続化要件

| ID    | 要件                                                                          |
| ----- | ----------------------------------------------------------------------------- |
| ST-01 | Index は標準ライブラリのみで読み書き可能な形式で永続化する（ファイル形式は設計書で確定。参考実装: JSON） |
| ST-02 | Index には chunk のテキスト・ベクトル・文書パス・見出し階層・更新日時を含める |
| ST-03 | 文書ごとにハッシュを保持し、差分更新時は変更検出に使用する（ハッシュアルゴリズムは設計書で確定。参考実装: SHA-256） |

### エラーケース

| 条件                               | 動作                                                                          |
| ---------------------------------- | ----------------------------------------------------------------------------- |
| OpenAI API key 未設定              | エラーメッセージで API key の設定方法を案内し、終了する                       |
| OpenAI API レート制限・通信エラー  | リトライし、最大試行後に該当 chunk をスキップして処理を継続。エラーログに記録 |
| 対象ディレクトリが存在しない       | エラーメッセージを表示して終了する                                            |
| 既存 Index と異なる Embedding 次元 | 既存 Index を破棄して再生成するか、ユーザーに確認する                         |
