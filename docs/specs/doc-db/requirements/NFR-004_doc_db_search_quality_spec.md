# NFR-004: doc-db 検索品質 要件

## 概要

doc-db の検索品質に関する要求を定義する。doc-advisor の ToC 方式を品質ベースラインとし、それと同等以上の recall を達成することを目標とする。

## 前提条件

- 評価機能（FNC-007）が動作する
- ゴールデンセットが利用可能である

## 要件一覧

| ID     | 要件                                                                                               |
| ------ | -------------------------------------------------------------------------------------------------- |
| QUL-01 | doc-db の recall は TBD-009 で定義する同等基準（ToC 方式の recall を基準とする）を満たす           |
| QUL-02 | false negative の傾向が分類され、原因（ID 検索失敗・長文希釈・自然文の意味ズレ等）が特定されている |
| QUL-03 | 結果は説明可能である（どの chunk・どのスコアでヒットしたかをユーザーが追跡できる）                 |
| QUL-04 | 検索品質改善のため、doc-db 内部実装の差分（chunk 粒度・スコア統合方式・rerank ON/OFF 等、具体的な比較軸は設計書 / 評価計画で列挙する）を定量比較できる |
| QUL-05 | ToC 方式 / doc-db 方式の採否を判断できる集計値が、ゴールデンセット評価で得られる                   |

## 品質目標の運用方針

- recall・precision は数値目標として運用するが、`spec_priorities_spec.md §3` に従い、構造品質（責務分割・境界）の数値化はしない
- 検索方式の良し悪しはゴールデンセット上の数値で判定し、見かけの達成率パイプラインを設計目標としない
