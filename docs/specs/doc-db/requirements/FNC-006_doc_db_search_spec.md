# FNC-006: doc-db 検索 機能要件

## 概要

ユーザーのクエリに対し、Embedding 検索と Lexical Search を組み合わせて候補を取得し、LLM Rerank で並び替えた最終結果を返す機能。

## 前提条件

- Index が生成済みである（FNC-005 完了済み）
- OpenAI API key が利用可能である

## 要件一覧

### 操作要件

| ID    | 要件                                                                                       |
| ----- | ------------------------------------------------------------------------------------------ |
| OP-01 | ユーザーは自然文または ID・固有名詞を含むクエリで検索できる                                |
| OP-02 | ユーザーは取得する結果数の上限（top-N）を指定できる                                        |
| OP-03 | ユーザーは検索方式（embedding のみ / lexical のみ / hybrid / hybrid + rerank）を選択できる |

### Embedding 検索要件

| ID     | 要件                                                                                                           |
| ------ | -------------------------------------------------------------------------------------------------------------- |
| EMB-01 | クエリを Index と同じ Embedding モデルでベクトル化する（モデルは設計書で確定。FNC-005 EMB-01 と一致させる）     |
| EMB-02 | クエリベクトルと chunk ベクトル間の意味的類似度で chunk を順位付けする（類似度尺度・探索アルゴリズムは設計書で確定。参考実装: cosine similarity / 線形探索） |
| EMB-03 | Multi-vector / Field 別ベクトルを持つ文書は、各ベクトルのスコアを集約してランキングに使用する（集約方式は設計書で確定） |

### Lexical 検索要件

| ID     | 要件                                                                                  |
| ------ | ------------------------------------------------------------------------------------- |
| LEX-01 | クエリ語の語彙一致に基づき、各 chunk のスコアを算出する（具体的なスコア式・パラメータは設計書で確定） |
| LEX-02 | 大文字小文字・全角半角・記号の正規化を行う                                            |
| LEX-03 | クエリに ID（例: `FNC-001`）や固有名詞が含まれる場合、完全一致を高スコアとして扱う    |

### スコア統合要件

| ID    | 要件                                                                                     |
| ----- | ---------------------------------------------------------------------------------------- |
| SC-01 | Embedding スコアと Lexical スコアを統合して候補をソートする                              |
| SC-02 | 統合方式は設計書で確定する。設定で切り替え可能とする（実験用途）                         |

### LLM Rerank 要件

| ID    | 要件                                                                               |
| ----- | ---------------------------------------------------------------------------------- |
| RR-01 | 統合スコア上位の候補を LLM に渡し、クエリとの関連度順に並び替える（モデル選定は設計書で確定。参考実装: OpenAI `gpt-4o-mini`） |
| RR-02 | LLM 呼び出しは外部 pip パッケージを必須としない手段で行う（HTTP クライアントは設計書で確定） |
| RR-03 | LLM Rerank は ON/OFF を切り替えできる（コスト・速度を優先する場合に OFF にできる） |

### 出力要件

| ID     | 要件                                                                                              |
| ------ | ------------------------------------------------------------------------------------------------- |
| OUT-01 | 結果は文書パス・該当 chunk の見出し階層・スコア・スコア内訳（embedding / lexical / rerank）を含む |
| OUT-02 | 結果は説明可能性のため、なぜヒットしたか（どの chunk のどのスコアで残ったか）を提示できる         |

### エラーケース

| 条件                    | 動作                                                                   |
| ----------------------- | ---------------------------------------------------------------------- |
| Index が存在しない      | エラーメッセージで Index 生成 skill の実行を案内する                   |
| OpenAI API 呼び出し失敗 | リトライし、最大試行後にエラーを返す。Embedding 失敗時は検索を中止     |
| LLM Rerank 失敗         | rerank をスキップし、統合スコア順の結果をそのまま返す（fallback 動作） |
