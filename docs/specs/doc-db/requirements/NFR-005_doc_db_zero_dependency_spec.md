# NFR-005: doc-db 配布制約 要件

## 概要

doc-db は配布の容易さを担保するため、ユーザー環境への追加要求を最小化する。具体的には次の 3 軸で配布制約を定義する: (1) Python 標準ライブラリのみで動作する（pip install 不要）、(2) 常駐サーバープロセスを必須としない、(3) Embedding / LLM 提供者以外の追加ベンダー API key を必須としない。既存の bw-cc-plugins 全体方針（`NFR-003 standard_library_preferred`）を継承する。

## 前提条件

- 実行環境に Python 3 がインストールされている
- ネットワーク経由で Embedding API および LLM API にアクセスできる

## 要件一覧

| ID     | 要件                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| DEP-01 | doc-db は Python 標準ライブラリのみで動作する（標準ライブラリの範囲は CPython 公式ドキュメントに従う） |
| DEP-02 | ユーザーに pip install を要求しない                                            |
| DEP-03 | 常駐サーバープロセス（外部 DB サーバー・検索エンジンサーバー等）を必須としない |
| DEP-04 | Embedding / LLM 提供者以外の追加ベンダー API key を必須としない（参考実装では OpenAI API key 1 種類で完結） |

> 永続化形式（JSON 等）、Lexical Search の実装方式、reranker の選定（自前 LLM Rerank / 外部 reranker API の採否）は設計書で決定する。本要件は「ユーザー環境への要求」のみを規定する。

## 制約の理由

- 配布形態を簡素化し、`/plugin install` だけで利用可能にする
- 既存 doc-advisor の `NFR-003 standard_library_preferred_spec` と整合させ、bw-cc-plugins 全体のポリシーを維持する
- 環境依存（外部 wheel のビルド・追加サーバーの起動・別ベンダー API key 取得）を排除する
