# FNC-007: doc-db 評価機能 要件

## 概要

ゴールデンセットを用いて doc-db の検索品質を測定し、ToC 方式・現行 Embedding 方式と比較できるレポートを生成する機能。

### 評価の目的（誰が何を判断するか）

- ユーザー / 開発者は、ToC 方式 / 現行 Embedding 方式 / doc-db 方式のうち、用途に応じてどの方式を採用するか判断できる
- 開発者は、false negative の傾向を把握し、検索品質改善の優先順位を決定できる
- 開発者は、doc-db の方式が doc-advisor 本体に取り込む価値があるかを判断できる

各評価指標・レポート項目はこれらの判断を支える従属要件として位置付ける。

## 前提条件

- ゴールデンセット（`meta/test_docs/`）が利用可能である
- ToC 方式・現行 Embedding 方式が同一ゴールデンセットで動作する状態にある
- doc-db の Index が生成済みである

## 要件一覧

### 操作要件

| ID    | 要件                                                                                                |
| ----- | --------------------------------------------------------------------------------------------------- |
| OP-01 | ユーザーは評価 skill を実行することで、ゴールデンセット全クエリの評価レポートを生成できる           |
| OP-02 | ユーザーは比較対象の方式（ToC / 現行 Embedding / doc-db）を選択できる（doc-db 内部の検索パラメータ比較軸は設計書 / 評価計画で定義する） |
| OP-03 | ユーザーは個別クエリの結果を確認でき、期待結果との差分を確認できる                                  |

### 評価指標要件

| ID     | 要件                                                                |
| ------ | ------------------------------------------------------------------- |
| MET-01 | 検索品質指標として recall・precision・false negative 数を測定する   |
| MET-02 | 性能指標として検索時間（クエリ → 結果）と Index 生成時間を測定する  |
| MET-03 | コスト指標として API 呼び出し回数と消費 token 数を測定する          |
| MET-04 | 結果の説明可能性として、ヒットした chunk の見出し階層を結果に含める |

### レポート要件

| ID     | 要件                                                                                                    |
| ------ | ------------------------------------------------------------------------------------------------------- |
| REP-01 | レポートは方式ごとの集計値（recall・precision・検索時間・コスト）を表形式で提示する                     |
| REP-02 | レポートは false negative の発生クエリと対象文書を一覧化し、傾向（ID 検索失敗・自然文失敗等）に分類する |
| REP-03 | レポートは Markdown 形式で出力し、人間と AI の両方が読める形式とする                                    |

### エラーケース

| 条件                         | 動作                                                           |
| ---------------------------- | -------------------------------------------------------------- |
| ゴールデンセットが存在しない | エラーメッセージでゴールデンセットの設置先を案内する           |
| 比較対象方式が動作しない     | 該当方式を結果から除外し、警告を表示してレポート生成を継続する |
| API レート制限による評価中断 | 中断時点までの結果を保存し、再開できるようにする               |
