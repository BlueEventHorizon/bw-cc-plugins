# REQ-005: auto-toc 要件定義書

| 項目     | 値                                   |
| -------- | ------------------------------------ |
| 要件ID   | REQ-005                              |
| 関連要件 | REQ-001（Doc Advisor コア要件）      |

## 概要

doc-advisor プラグインに、Claude Code セッション終了時に ToC（Table of Contents）を
スクリプトベースで自動更新する機能（auto-toc）を追加する。
既存の AI ベース ToC 生成（create-rules-toc / create-specs-toc）と共存し、
自動更新とリッチメタデータ生成の 2 層構造を形成する。

## 前提条件

- `.doc_structure.yaml` がプロジェクトルートに存在し、rules/specs ディレクトリが設定済みであること
- doc-advisor プラグインがインストールされていること
- `plugin.json` が Stop フックの定義に対応していること（TBD-003 参照）

## 機能要件 [MANDATORY]

### FNC-001: Stop フック自動トリガー（優先度: Must）

- セッション終了時（Stop フック）に自動的に ToC 更新処理が起動する
- ユーザーによる手動操作は不要

### FNC-002: 差分検知（優先度: Must）

- セッション内で変更（追加・編集・削除）されたファイルのみを検出する
- 変更ファイルが rules/specs ディレクトリ配下に存在しない場合は何もしない
- `.doc_structure.yaml` の `scan.exclude` に該当するファイルは検出対象外とする

### FNC-003: カテゴリ別更新（優先度: Must）

- 変更ファイルが rules 配下 → rules ToC を更新する
- 変更ファイルが specs 配下 → specs ToC を更新する
- 両方に変更あり → 両方を更新する

### FNC-004: スクリプトベース処理（優先度: Must）

- auto-toc の更新処理は AI を呼び出さずスクリプトのみで完結する

### FNC-005: 既存スキルとの統合（優先度: Should）

- `create-rules-toc` / `create-specs-toc` スキルから auto-toc スクリプトを呼び出せる
- 受け入れ条件: `create-rules-toc` または `create-specs-toc` の実行時に auto-toc スクリプトが呼び出されること
- 統合方法（オプション追加またはスキル改修）は設計フェーズで決定する

### FNC-006: プラグインフック統合（優先度: Must）

- `plugin.json` の `hooks` フィールドで定義し、プラグインインストール後から自動適用される
- ユーザーによる `settings.json` の手動設定は不要

### FNC-007: エラーハンドリング（優先度: Must）

- スクリプト実行が失敗した場合（異常終了コード）、エラー内容を標準エラー出力に出力し、セッション終了を妨げない
- `.doc_structure.yaml` が存在しない、または破損している場合はスキップする（エラーにしない）
- ToC ファイルへの書き込みが失敗した場合（権限エラー等）、エラー内容を通知し、既存 ToC を破壊しない
- ToC ファイルが存在しない初期状態（初回セットアップ直後）では、auto-toc は更新をスキップする（新規作成は `create-rules-toc` / `create-specs-toc` の責務）

## 非機能要件 [MANDATORY]

### NFR-001: 実行通知（優先度: Must）

- ToC 更新が実行された場合、更新対象カテゴリ・完了件数を1行で標準出力する
  - 例: `[auto-toc] rules: 2 files updated`
- 変更なしの場合は何も表示しない

### NFR-002: 互換性（優先度: Must）

- auto-toc が更新した ToC に対して、既存の `query-rules` / `query-specs` が ToC の YAML スキーマを正しく読み取れること
- AI メタデータ（keywords, applicable_tasks 等）の品質は TBD-001 の解決策に依存する

### NFR-003: 実行速度（優先度: Should）

- スクリプトの実行時間はセッション終了を遅延させない程度に短時間で完了すること
- 具体的なタイムアウト値は設計フェーズで決定する
