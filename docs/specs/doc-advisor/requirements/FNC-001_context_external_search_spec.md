# FNC-001 コンテキスト外検索 要件定義書

## 概要

doc-advisor の文書検索（query-specs / query-rules）において、検索処理を AI のコンテキストウィンドウの外で実行し、AI には検索結果（候補パスのリスト）のみを渡す。

`query-specs` / `query-rules` SKILL は `context: fork` の subagent として実行され、subagent は fork 内で検索を完結させ、最終結果のみを親 Claude に return する（subagent 内完結フロー、DES-006 §2.4 参照）。

## 前提条件

- doc-advisor プラグインがインストールされていること
- `.doc_structure.yaml` が設定済みであること
- 対象文書がスキャン対象ディレクトリに存在すること
- OpenAI API key が環境変数で利用可能であること（`auto` モード / `--index` モード時）。参照規約は FNC-004 / DES-007 を参照

## 要件一覧

### 機能要件

- `--index` モードでは、文書の検索処理が AI のコンテキストウィンドウを消費せずに実行される（`search_docs.py` / `grep_docs.py` による doc-advisor Embedding 検索）
- `auto` モードでは、doc-db plugin が利用可能なら `/doc-db:query --mode rerank` による **Hybrid 検索（Embedding + Lexical + LLM Rerank）** が AI のコンテキスト外で実行される。doc-db 未インストール時は ToC キーワード検索へ自動フォールバックする
- `--toc` モードおよび `auto` モードの ToC フォールバック時は、AI が ToC YAML を Read してキーワードマッチングを行う（コンテキストを消費するが、高精度）
- 検索結果は候補文書のパスリストとして AI に渡される
- AI は受け取ったパスリストの文書を Read して内容を確認する
- query-specs と query-rules の両方で同じ 3 モード検索方式が使用される
- ユーザーから見た実行方法と出力形式は変わらない（モード切替は `--toc` / `--index` オプション、省略時は `auto`）

### 検索の入力

- タスクの説明文（自然言語テキスト）

### 検索の出力

- AI に渡す最終出力: 候補文書のパスリスト（プロジェクトルート相対パス）
- 検索エンジン内部レスポンス: パスリストに加え、スコアまたは関連度情報を含む（内部利用、AI には渡さない）

### テスト要件

- 新規に作成する Python スクリプトには `tests/` 配下に自動テストを追加する

### エラーケース

| 条件                                              | 動作                                                                                                                 |
| ------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| 検索インデックスが存在しない                      | インデックスの生成を案内する（`auto` モードでは doc-db Index 未構築時に自動ビルドを実行）                            |
| 検索クエリが空                                    | エラーメッセージを返す                                                                                               |
| 該当文書が 0 件                                   | 空のリストを返す                                                                                                     |
| `auto` モード時に doc-db plugin が未インストール  | ToC キーワード検索へ自動フォールバックする                                                                           |
| `auto` モード時の doc-db 検索失敗（API エラー等） | error + hint を通知して停止する（ToC へ自動フォールバックしない。検索品質劣化の可視性確保のため、DES-006 §2.3 参照） |
