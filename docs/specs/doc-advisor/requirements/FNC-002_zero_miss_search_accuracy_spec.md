# FNC-002 見落としゼロの検索精度 要件定義書

## 概要

doc-advisor の文書検索において、関連する文書を 1 件も見落とさない検索精度を実現する。返却する文書の件数に上限を設けず、必要な文書は全て返す。

## 前提条件

- FNC-001（コンテキスト外検索）が実装されていること
- 検索インデックスが最新の状態であること

## 要件一覧

### 精度要件

- 関連する文書は全て検索結果に含まれる（False Negatives = 0）
- 件数制限を設けない。関連文書が 50 件あれば 50 件全て返す
- 現行の AI による ToC 全量検索と同等以上の精度を達成する

### 検索対象のフィールド

- タイトル（title）
- 目的（purpose）
- 内容詳細（content_details）
- 適用タスク（applicable_tasks）
- キーワード（keywords）

### カテゴリ分離要件

- 現行の rules / specs のカテゴリ分離を維持する
- 各カテゴリのインデックスは独立して生成・検索される

### 精度検証要件

- 精度を測定するための固定テストクエリセット（ゴールデンセット）を用意する
- ゴールデンセットの正解データは人手で確認・作成する（現行 AI 検索の結果は初期ベースラインとして使用し、人手レビューで補正する）
- 新方式の結果とゴールデンセットを比較し、見落とし件数を測定する
- 見落とし 0 件を確認してから本番適用する

### 品質テスト結果（v0.2.0 実施）

v0.2.0 でゴールデンセット（43 クエリ: rules 21 件 + specs 22 件、6 種別: direct/task/crosscut/synonym/proper/negative）を使用した品質テストを実施:

- **ToC（`--toc`）**: 最も精度が高い。AI による YAML メタデータ全量マッチングにより false negative が最小
- **Index（`--index`）**: 良好な精度。ToC メタデータが古い文書で ToC が見落とすケースを補完
- **auto（ハイブリッド）**: 最良の網羅性。両方の候補を union し AI が最終判定

テスト基盤: `meta/test_docs/` に配置（git 管理外）

### エラーケース

| 条件 | 動作 |
| ---- | ---- |
| インデックスが古い（文書変更後に未更新） | 警告を表示し、インデックス再生成を案内する。再生成完了までは検索を実行しない（古いインデックスでの検索は見落としリスクがあるため） |
