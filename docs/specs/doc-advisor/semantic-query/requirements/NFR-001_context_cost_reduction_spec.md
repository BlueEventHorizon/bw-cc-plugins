# NFR-001 コンテキストコスト削減 要件定義書

## 概要

文書検索時に AI のコンテキストウィンドウに読み込まれるデータ量を削減する。現行の ToC YAML 全量読み込みを不要にする。

## 前提条件

- FNC-001（コンテキスト外検索）が実装されていること

## 要件一覧

### コスト削減要件

- 検索時に ToC YAML（現行 850KB 規模）を AI のコンテキストに読み込まない
- AI が消費するコンテキストは、検索結果（候補パスリスト）と候補文書の本文のみとなる
- コンテキスト消費量が現行比 50% 以上削減される

### 測定方法

- 測定単位: トークン数（AI のコンテキストウィンドウ消費量）
- 測定条件: 固定クエリセット（FNC-002 のゴールデンセットと共通）での平均値
- 現行: ToC 読み込みトークン数 + 候補文書の本文読み込みトークン数
- 新方式: 候補パスリストのトークン数 + 候補文書の本文読み込みトークン数
- 差分が 50% 以上であることを確認する

### エラーケース

| 条件 | 動作 |
| ---- | ---- |
| 検索インデックスが利用不可 | ToC YAML へのフォールバックは行わない（FNC-003 のエラーケースに従う） |

## 関連要件

- FNC-001: コンテキスト外検索
- NFR-002: YAML ToC の廃止
